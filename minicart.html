Mini Cart

Re-designing the mini-cart to introduce common design patterns that help build user confidence and reduce cart abandonment.

TL;DR

The previous product designer that worked on the portal had introduced a 'multiple mini-cart' feature, which was an uncommon design pattern. Common design patterns for a mini-cart were not being used, which I suspected could be impacting on the usability of the mini-cart and contributing to cart abandonment or drop off where the user was adding to the mini-cart but not moving into the shopping cart page.

The approach I adopted was to firstly undertake some research to gather data that could be used to inform my next actions. I researched other e-commerce products, gathered quantitative data from the analytics tool 'Heap' and also spoke to users to back this data up with their qualitative feedback.

Data Driven Design

Quantitative Research - Engagement with Existing Design

I wanted to ascertain whether features introduced with the existing multiple mini-cart were being used. One of the features was a 'Create Cart' button that enabled a user to create multiple mini-carts. Speaking to the product designer who worked on this previously he advised that the idea behind the design was to enable partners to add products to the mini-cart for a particular customer and then enable them to create a new mini-cart for another customer where they could again add more products. They would then complete the user buying journey for each customer.

This design pattern was something not common to mini-carts and seemed to be a bit complex, so I was interested to see if it was being adopted by our partners. I began by gathering data on the existing design. Analytics from 'Heap' showed that there was no engagement with the 'Create Cart' button. Speaking to partners about this feature they also advised that they did not use it. So as expected this uncommon design pattern had not been adopted, so needed to be changed.

The analytics graph from 'Heap' showed no engagement on the 'Create Cart' button and almost no engagement on the 'Select Cart' button (this button allowed partners to move between carts). This data proved that these features were not used. There was engagement on the 'Review Cart' and 'Delete' icon buttons, so these features would be included in the new design of the cart.

Engagement on 'Multiple Cart' Design

Online Research - Identifying Common Design Patterns

From my research I was able to quickly identify design patterns that most e-commerce products used for their mini-carts. These included:

  1. Indicating the number of items that are in the mini-cart.
  2. Opening the mini-cart whenever the user clicks on 'add to cart'.
  3. Displaying the number of items in the cart, along with the total price.
  4. Allowing the user to delete each item individually from the mini-cart.
  5. Quantity adjustments from within the mini-cart to upsize or downsize to the item.

Proposed Design - Stage_1 Improvements

I discussed improvements with the development team and they advised that they could implement the needed changes in two stages as there were already a lot of commitments involved in upcoming sprints. The first stage would remove the 'multiple cart' feature and add an indicator of items to the cart, improve the usability by allowing the user to click from the mini-cart into the shopping cart and also opening the mini-cart whenever a user added a product.
