Jiggle

A vibrant and modern music player app.

My role: UI Design

01

Overview

'Jiggle' is a music player app.

This was the first project undertaken as part of the UI Design course. During the development of the project a number of topics were studied. These included:

  • Design Principles & Heuristics.
  • Design Patterns.
  • Wireframing.
  • Layout and Spacing.
  • UI Elements & Hierarchy.
  • Iconography.
  • Text & Typography.
  • Use of Colour in UI.

Scroll To Top

Objective

Develop a mobile app having first researched and identified UI design patterns that will be used in the app.

Key features proposed:

  • Home screen - Be able to view recently played and popular tracks.
  • Song Screen - Be able to shuffle, play, pause, repeat, add to playlist, add to favourites.
  • Search Screen - A search form with autocomplete functionality, card UI elelments of music categories.
  • Library Screen - Tabbed horizontal navigation, search form, sorting functionality.
  • Playlist Screen - Add songs to playlist, download songs, search in playlist functionality.

02

Sketching

Initially low fidelity wireframes were created using pen & paper. Only the essential content and functionality of the app was considered at this stage without going into too much detail. Mobile sketching templates were downloaded and used as can be viewed below. At this point UI design patterns that were researched were sketched into the proposed design of the app. Annotations were then added to indicate what was to be included in each screen.

03

Digital Wireframes

Following on from the paper sketches medium fidelity wireframes were developed using Sketch. As this was one of the first student projects a number of new concepts were introduced. Initially placeholder Icons were used for the wireframes that were downloaded from the web but later these Icons were created within Sketch itself. The icon family that was used for the main navigation was designed using a speaker symbol to tie in with the name and brand logo and to give the icon family a consistent feel. Creation of various UI elelments, spacing and alignment using the grid system, and visual hierarchy were all considered at this stage of the design.

04

Inspiration Board

The next step of the project was to decide on a clear visual style. A look and feel for the app needed to be created. Use of color to evoke emotions through the UI was considered. Online resources were viewed in order to obtain inspiration for colours, textures, gradients, typography, imagery, quotes and wording. Once content had been obtained the most relevant was gathered together and displayed on an inspiration board as seen below. The feel for the app was a youthful, vibrant, happy and energetic.

05

High Fidelity Wireframes

I decided to use an orange colour as the dominant colour for the app as I wanted to create a feeling of warmth, energy and fun which is suitable for the product as it is created for the music industry and is intended to appeal to a young audience. Yellow is used as a support colour as yellow sits next to orange on the colour wheel and they naturally blend together. Yellow is associated with happiness and enforces the feeling created by orange. Blue is used as the accent colour as it sits opposite orange on the colour wheel so gives the project some ‘pop’ when used sparingly and helps direct the users eye to elements in the interface. The colder shade of the blue helps emphasize the warmth of the orange. A dark grey colour is used for the font as it is more pleasant on the eye to read but also helps to act as a contrast for the bright orange colour, and is used for the button Icons on the footer when not in an active state. The slightly off white colour is used to give the project a clean fresh look when used as a background and displayed against the warm orange and yellow colours.

Style Guide

07

Retrospective

What went well?

I was peased with the development of the wireframes from initial sketching through to high fidelity wireframes. As this was my first app developed using Sketch there was a learning curve with using the software that I overcame and I learnt a lot about design principles as well. The Icon family that I created for the project turned out well, I think the use of the speaker symbol in the Icons helps re-enforce the brand and gives the app a unique look. I also think the use of the yellow/orange gradient works well to give the app a vibrant feel and also the use of gradients are currently in trend.

Skills:

Sketching, Wireframes.

Processes:

Design principles & trends.

Learnings:

Continue to research UI design patterns and trends and take inspiration from online sources.

What was a challenge?

There was a learning curve involved with the Sketch software. In particular I found the use of the grid system to be a bit tricky at first and this led to a number of iterations before I had the right grid in place. Creating my own Icons for use in the app was another aspect that was inititally time consuming and involved practice with the software.

Skills:

Sketch software.

Processes:

Grids & UI elements.

Learnings:

Continue to build experience with using the software.
