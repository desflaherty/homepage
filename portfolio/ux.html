Fly UX

A user-friendly flight booking desktop website.

My role: UX Research & UX Design

01

Overview

'Fly UX' is a flight booking website for desktop. The study involved conducting research on existing flight booking websites to identify pain points users were experiencing, and use the findings to design a new solution that would overcome these frustrations and lead to a better user experience in the flight booking process.

The prototype was created following a user centered design process including:

  • Discovery - Research, Moderated Usability Testing, Depth Interviews, Competitor Analysis, An Online Survey, Data Insights.
  • Concepting - Affinity Map, Customer Journey Diagram
  • Design - Desktop Flow Diagram, Define Navigation, Desktop Interaction Design.
  • Prototyping & Testing - Prototype Build, User Testing.

02

Discovery

Moderated Usability Tests

The first assignment undertaken was to conduct two moderated usability tests to obtain some qualitative data about the usability of existing airline websites. I began by defining my research objectives to determine what I wanted to achieve from the tests. I created a recruitment screener, a consent form and a test script to use during the desktop usability tests.The test script was designed ensuring not to use leading or closed questions as these would impact the quality of the data obtained. I conducted two desktop usability tests using two different test subjects on the Aer Lingus and Ryanair websites.

Test Objectives:

  1. Identify typical user behaviour on the Aer Lingus & Ryanair websites when booking a flight.
  2. Discover the primary goals of the user.
  3. Understand 'best in class' features being used by the airlines.
  4. Understand features that cause the user to be frustrated.
  5. Identify pain points that could cause the user to leave the website.
  6. Identify features and areas on the website that could be improved for better UX.

Competitor Analysis

A competitor analysis was then undertaken. I reviewed four websites to identify what best in class competitor websites were currently doing well. I wanted to gain more insight into how these websites were solving problems that users face when booking flights, understand current conventions that should be followed and highlight best practice being used.

Online Survey

A survey was conducted using Survey Monkey to gather more qualitative and quantitative attitudinal research to gain more understanding into context of use. I used a mixture of open and closed questions as well as some multiple choice questions. In keeping with good practice the survey contained no more than 10 questions and would take around 2-3 minutes for people to complete. Some of the survey responses can be viewed on the graphs below.

03

Concepting

Affinity Map

The usability tests were reviewed and the recorded notes on the test sessions were examined. I then wrote down every important point that was mentioned onto separate post-it notes. The depth interview, competitor analysis and data gathered from the online survey was also reviewed and post-it notes were developed from this research also. The notes were stuck onto a whiteboard and organised into categories. These categories were used to create the Affinity Map as shown below.

Customer Journey Diagram

The next step was to use the insights obtained from the Affinity Map to create the Customer Journey diagram. I reviewed the insights and organised them into a step by step view of the customer journey through the airline websites to demonstrate how the customer felt at each stage of the process. I noted the customers goals, behaviours, context and pain points at each step of the journey. Icons were used to demonstrate how the customer felt during the process, as to whether they were happy, frustrated or upset.

Data Insights

From the work completed up to this point some insights emerged regarding users frustrations and pain points.

# Users Expectations & Pain Points
1 Expected their location (i.e Dublin) to be populated automatically for their departure point.
2 Experienced frustrations around calendar functionality and flight searches.
3 Didn't want to have to re-type personal data all the time on forms.
4 Wanted more clarity around the price being paid for the flights before they reached the checkout point in the process.
5 Wanted more helpful information at each stage in the booking process.
6 Were being frustrated by the addition of add ons and upsell messages being displayed before checkout.
7 Wanted a guest checkout option, as they didn't want to be forced to login or sign up in order to purchase the flight.

04

Design

Flow Diagram

I defined a high level flow for the desktop website with the objective of fixing any issues that I would have discovered during the research phase and highlighted on the Customer Journey diagram. The task for the project was to focus on one flow or primary use case. I initially sketched out the diagram before using Adobe Illustrator to create the flow.

Define Navigation

I began sketching out some initial concepts for the airline navigation. On the homepage there would be primary and secondary navigation menus that would be hidden using a hamburger menu icon once the user has searched for flights. The hidden menu links would be available through the booking process until the user enters the checkout screens where a status bar would display the steps needed to complete the payment process. There would be an easy option to skip all add ons in order to complete the process as quickly as possible.

Desktop Interaction Design

My objectives for this part of the process were to sketch the screens and the screen states for the users flowing through the flight booking website addressing all the issues and user goals identified previously in my research. Interactions would have included predictive fields where the airport name populates when three characters have been entered into the search form, the cheapest price for a flight being clearly highlighted, being able to use the arrow buttons to search for a new flight without errors appearing, a simple skip button to avoid add on screens, a guest checkout option and a total fare with cost summary being displayed before checkout.

05

User Testing

I designed each screen using Sketch in Medium Fidelity form to create a prototype. This prototype could then be tested and further developed before creating the wireframes. I ensured that the prototype contained enough detail and interactivity to test the high level flow, the screen layouts, text and basic interactions. The finished prototype was brought into InVision to test the interaction design.

The original participants that were used for the initial usability tests that identified pain points with existing airline websites were again contacted and asked to test the new prototype.

There was also some constructive criticism that was taken on board and led to some changes in the UX design:

The confirmation screen before testing offered the user two choices - 'Book New Trip' and 'Sign Up'. The user feedback was that they wanted an option to end the session as they might not want to sign up or to book a new trip. Feedback was also that the users would like to see a summary of their booking on the confirmation page so that they could be sure everything was correct. Also the option to sign up at this point was not something that the users stated they would use, as they already had this option during the flight booking process and prefered to use the guest checkout option instead.
