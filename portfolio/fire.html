Dublin Fire Brigade

Interactive Dashboard

The Brief

Develop the interface for an interactive dashboard and publish it to the web. Use open source data avaialble online for the visualization.

Concept Development

The intention of the project would be to use custom visualizations in order to provide insight on the incidents reported by the fire services in the Dublin area over a 3 year period from 2013-2015. This project is intended to provide insight into this area for public consumption as it showcases the work being done by the emergency services to ensure the safety of the public in Dublin. Insights gained from using the dashboard could highlight peak times of the day for call outs, problem times of the year i.e Halloween, areas with highest call out frequency, and additional special requests that DBF provide.

UX

User stories were first developed to determine what the function of the dashboard should be for a positive user experience. It was determined that the visualization should allow the user to

  • Use filter menus to navigate the dataset,.
  • Filter the recipes by searching using different criteria such as Cuisine, Course, Occasion, Special Diet.
  • View a total amount of call out incidents contained in the dataset.
  • Have a graphical display of total call outs by area.
  • View which areas and incidents are generating the most call outs.
  • View a time line of call out activity to highlight spikes in the year such as Halloween.
  • Have a view of peak time and day of calls.
  • Be able to interact with these visualizations by filtering on different areas of interest that would help generate insight.
