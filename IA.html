Information Architecture

Using data analysis to re-design the information architecture of the platform including the main navigation elements on the homepage.

TL;DR

I analysed the homepage navigation elelments to determine which elements were being used mostly by the partners. I was hoping to be able to use this analysis to make improvements to the IA. The ONEClub portal was linked to an analytics platform called 'Heap' and also to another tool for analysing customer click patterns called 'Hotjar'. I reviewed the data from these tools and created mockups to improve the information architecture, reducing the number of clicks needed to access the most used navigation elements on the homepage while also reorganising the content within the platform.

Data Analysis

1) Updating the 'Sell' label on the navigation menu

I created custom events in 'Heap Analytics' to count the unique number of users that interacted with the main navigation menu on the homepage in the last 30 days. This analysis showed that the 'Subscriptions' link was being used mostly along with the 'Home' link. The 'Buy' link had the next highest engagement, followed by the 'Reports'. The 'Sell' link (which navigated users to 'Customers' and also to the 'Price Lists') had the lowest engagement.

I also looked at the interactions on the main navigation on the homepage using a heatmap that I created in 'Hotjar'. The heatmap recording backed up the data analysis from 'Heap' as can be seen in the image below where the click activity by users is shown. 'Subscriptions' has the largest heat signature indicating the most clicks, followed by the 'Buy' link, then 'Reports' and lastly the 'Sell' link.

I continued the analysis by looking at the user interactions with the main card elements on the homepage, these cards also acted as links to the different areas on the portal. Interestingly the card that was being interacted with the most by partners was the 'View Customers' card. This card had an engagement rate of 67.2% in the last 30 days when compared to the other navigation cards shown here on the homepage.

Going back to look at the least used link on the main navigation (labeled 'Sell'), I could see that there were two cards within this section. One of these was a 'Manage Customers' card and the other was a card to link to 'Price Lists'. The 'Manage Customers' card was another way that users could navigate to their customer list, but it was hidden inside of the 'Sell' section .

It became obvious that users were using the 'View Customers' card on the homepage as the main way to navigate to their customers and that the 'Sell' link on the main navigation was not being used as a navigation route to customers because the label name had no meaning. I decided that an improvement to the navigation would be to rename the 'Sell' link on the navigation menu to 'Customers' and also to remove the section showing the two cards. 'Manage Customers' would no longer be needed, and 'Price Lists' could be moved to another section of the portal. The main objectives were to make customers more accessible for partners by renaming the label on the navigation menu and reducing the number of clicks needed to access their customer list.
